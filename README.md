# cv-lending-api

Backend/API часть проекта `cv-lending`.

API реализовано на Express и отвечает за:

- приём данных из контактной формы;
- валидацию входящих данных;
- отправку email через SMTP;
- отправку копии письма пользователю;
- генерацию AI-summary для обращения;
- обработку ошибок внешних сервисов;
- защиту от частых запросов через rate limit.

---

## Стек

- Node.js
- Express
- TypeScript
- Nodemailer
- CORS
- express-rate-limit
- dotenv
- OpenRouter API / совместимый AI Chat Completions API
- Vercel для деплоя

## Как запустить проект

### 1. Установить зависимости

```bash
npm install
```

### 2. Создать `.env`

Создайте файл `.env` на основе `.env.example`.

Пример:

```env
NODE_ENV=development
PORT=4000

CORS_ORIGIN=http://localhost:5173

RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX=100

SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
MAIL_FROM=your-email@gmail.com
MAIL_TO=owner-email@example.com

AI_API_KEY=your-api-key
AI_MODEL=openai/gpt-oss-120b:free
AI_PROVIDER=ai-provider-url
```

### 3. Запустить dev-сервер

```bash
npm run dev
```

По умолчанию API будет доступно по адресу:

```txt
http://localhost:4000
```

## Эндпоинты

### `GET /health`

Проверка статуса API.

Пример ответа:

```json
{
  "status": "ok"
}
```

---

### `POST /api/contact`

Принимает данные контактной формы с frontend.

Пример запроса:

```json
{
  "name": "Иванов Иван",
  "phone": "+799 999 555 35 35",
  "email": "ivan@example.com",
  "comment": "Я хочу обсудить проект по фронтенду."
}
```

Что делает endpoint:

- валидирует данные формы;
- формирует письмо владельцу сайта;
- отправляет письмо на `MAIL_TO`;
- отправляет копию письма пользователю;
- возвращает результат отправки на frontend.

Если SMTP-переменные не настроены, API может работать в демонстрационном режиме: запрос проходит валидацию, но письмо фактически не отправляется.

---

### `POST /api/ai-summary`

Endpoint для AI-summary обращения пользователя.

Пример запроса:

```json
{
  "prompt": "React-лендинг с формой обратной связи и отправкой email"
}
```

Что делает endpoint:

- принимает текстовый prompt;
- отправляет запрос в AI API;
- получает краткое summary;
- возвращает результат на frontend.

Пример ответа:

```json
{
  "summary": "Пользователь хочет обсудить разработку React-лендинга с рабочей формой обратной связи и email-уведомлениями."
}
```

Если AI API недоступен, возвращает fallback-ответ или ошибку, при этом основная логика контактной формы не должна ломаться.

## Как реализована форма

Форма реализована по схеме:

```txt
Frontend → POST /api/contact → Backend validation → SMTP → Email owner + Email copy to user
```

Backend отвечает за критичную серверную логику:

- не хранит SMTP-данные на клиенте;
- не раскрывает API-ключи во frontend;
- проверяет входящие данные;
- обрабатывает ошибки отправки email;
- возвращает понятный результат для UI.

На frontend для формы предусмотрены состояния:

- `loading`;
- `success`;
- `error`;
- блокировка повторной отправки во время запроса.

## Обработка ошибок

В API предусмотрена обработка типовых ошибок:

- невалидные данные формы;
- ошибки SMTP;
- недоступность AI API;
- превышение лимита запросов;
- ошибки rate limit в production-окружении.

---

## AI-интеграция

В проект добавлена AI-функция генерации summary обращения.

AI используется как дополнительная возможность, а не как обязательная часть отправки формы. Если внешний AI-сервис вернёт ошибку или будет временно недоступен, основная отправка формы должна продолжать работать.

Для AI-интеграции используется OpenRouter API или совместимый Chat Completions API.
